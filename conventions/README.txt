This directory contains conventions for various types of files.
Please refer to these guidelines as needed so that the code in this project is consistent and readable.

General Principles:
	In general, favor clarity over conciseness. It's better to have a very long, descriptive variable name than a concise, esoteric one.
	Test everything. When possible, you should use Test Driven Development to ensure maximum automated test coverage.

Copyright and Contributors:
	Each non-binary file should include the copyright information above, as well as the creator of the file and a list of anyone who has contributed

Contribution:
	Each non-binary file should include the creator of the file, as well as any contributors.
